What "can log in" really depends on

A website can never spontaneously see that you're signed into a Google, Microsoft, or Apple account elsewhere — that would be a serious privacy leak. What it can do is ask, through each provider's real sign-in flow, and get a real yes-or-no answer. Whether that flow succeeds here depends on a few things stacking up correctly:

  • An app registration exists. Every provider requires a Client ID tied to a specific domain before it will show a login prompt to that page at all.
  • The page is served securely. All three providers require a real https address (or localhost) — opening this file directly from disk will block every one of them.
  • Popups and cookies are allowed. Sign-in windows are popups; if a browser or extension blocks popups or first-party cookies, the flow can't complete even with valid credentials.
  • Third-party cookie policy varies by browser. As of 2026, Safari and Firefox block third-party cookies by default, while Chrome still allows them but is shifting toward letting people choose. This mostly affects silent "stay signed in" checks, not the popup flows this page uses, which rely on first-party cookies during the popup itself.
